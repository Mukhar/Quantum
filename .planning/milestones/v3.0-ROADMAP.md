## Overview

v3.0 paired the canonical quantum-algorithm canon with the practical
use case each algorithm unlocks. Every v3 essay teaches the algorithm
AND the use case in the same scroll, with widgets driving both halves.
This closes the v1+v2 reader's last open question — *"Where does
this actually bite reality?"* — without compromising the
static-site, no-backend posture.

Phase numbering was reset to 1 for v3 (matching v2's pattern). v1
phases 1–5 archived under `.planning/phases/_archive-v1/`; v2 phases
1–4 under `.planning/phases/_archive-v2/`. v3 phases stay under
`.planning/phases/0X-*` until the next archival pass.

Foundation-first, ascending difficulty: infra → flagship → arc-close
→ self-contained → densest split into halves → variational + launch.

## Phases

### Phase 1: Foundation — Qiskit export + bundle CI

**Goal:** Land the two cross-cutting infrastructure pieces every later
phase depends on: a Qiskit-text export usable from both the sandbox
toolbar and every essay's `CircuitView`, and a per-route bundle-size
CI gate that prevents v3 essays from bloating the site.
**Depends on:** v1.0 codec (`src/lib/quantum/codec.ts`), v1.0
`CircuitView` component, v2.0 sandbox toolbar.
**Requirements:** QSK-01, QSK-02, QSK-03, OPS-04
**Plans:** 4 plans

- [x] 01-01: Qiskit exporter module + drift-proof tests (`src/lib/quantum/qiskit.ts`)
- [x] 01-02: Sandbox toolbar "Copy as Qiskit" button
- [x] 01-03: `CircuitView` "Copy as Qiskit" button
- [x] 01-04: Per-route bundle-size CI gate (`scripts/check-bundle-budget.mjs`)

**Details:**
"Copy as Qiskit" button on sandbox toolbar emits a runnable
`QuantumCircuit(n,n)` snippet (gates in `Step` order + measurement).
Every essay-embedded `CircuitView` exposes the same affordance for
its statically-encoded circuit. Golden test asserts each
simulator-supported gate (`H`, `X`, `Y`, `Z`, `S`, `T`, `Rx`, `Ry`,
`Rz`, `CNOT`, controlled rotations, `SWAP`, `M`) maps to
syntactically-correct Qiskit and fails on any new simulator gate
without a Qiskit mapping. CI runs `astro build` and asserts each
tracked route's JS bundle stays under its declared ceiling
(`bundle-budget.json`); build fails on overrun. v1 + v2 tests all
still pass.

### Phase 2: Teleportation + Quantum networks (FLAGSHIP)

**Goal:** Ship `/teleportation`, the flagship v3 essay, proving the
"algorithm half + use-case half, both widget-driven" format

teleportation circuit; use-case half lets them swap entanglement
across a 3-node network.
**Depends on:** Phase 1 (Qiskit export is on this essay's
`CircuitView` from day 1), v1 `MiniBloch` pattern,
`src/lib/quantum/reducedDensity.ts`.
**Requirements:** ALG-01, ALG-02, USE-01
**Plans:** 6 plans

- [x] 02-01: `quantum/teleportation` module + correctness tests
- [x] 02-02: Shared `ProtocolStepper.astro` component
- [x] 02-03: `MultiBlochPanel.astro` (composes 3 × `MiniBloch`)
- [x] 02-04: Generic `QuantumNetwork.astro` widget (USE-01)
- [x] 02-05: `/teleportation` essay + all mirrors + bundle-budget entry
- [x] 02-06: Bundle-ceiling recompute close-out

**Details:**
`/teleportation` essay renders end-to-end with an embedded
`ProtocolStepper` that advances through entangle-pair → Bell measure
→ classical bits → conditional X/Z → state arrives. `MultiBlochPanel`
honestly renders mixed states: Bloch arrow length = `|r|` (vector
norm of reduced density); for canonical mixed states (`I/2`,
`(I + 0.5·Z)/2`) the arrow lives **inside** the sphere with the
right magnitude. `QuantumNetwork` 3-node widget (Alice / Repeater /
Bob): clicking each link swaps entanglement along it; live "shared
Bell pair Alice ↔ Bob" indicator flips when end-to-end entanglement
exists. Essay frontmatter wires into `tests/essays/nav-graph.test.ts`
mirror; concept-map node added with same-commit mirror update.

### Phase 4: Grover + Search reality

**Goal:** Ship `/grover`. Algorithm half implements oracle + diffusion

half is the "honest disappointment" — quantum search is `√N`, not
magic.
**Depends on:** Phase 1 (Qiskit export); independent of Phases 2-3.
**Requirements:** ALG-04, USE-03
**Plans:** 6 plans

- [x] 04-01: `quantum/grover` module (oracle + diffusion + iteration count)
- [x] 04-02: `AmplitudeBars.astro` widget (per-basis-state amplitudes)
- [x] 04-03: `SearchComparison.astro` widget (classical vs Grover)
- [x] 04-04: `/grover` essay scaffold + structure
- [x] 04-05: `/grover` essay body + RSA-handoff prose
- [x] 04-06: Mirrors + bundle-budget entry

**Details:**
Oracle + diffusion operators implemented against the existing
simulator; unit tests assert amplitude concentration on the marked
state after the optimal `⌊(π/4)·√N⌋` iterations. `AmplitudeBars`
widget shows per-basis-state amplitudes after each iteration; reader
steps through; bars animate. `SearchComparison` widget side-by-side
animates classical linear scan vs Grover's `√N` as `N` slides from
16 → 1024. Body text explicitly closes the "does this break RSA?"
loop with a clean "no — that's Shor, see next essay" handoff.

## Milestone Summary

**Decimal Phases:**

- Phase 5a: Shor — QFT + period-finding (split from original Phase 5
  to keep each plan reviewable)
- Phase 5b: Shor — N=15 + RSACountdown + PQC (depends_on = 5a, same
  `/shor` URL extended)

**Key Decisions (ratified across v3 phases):**

- **Interleaved essays (algorithm + use case)** — Deepens v1's
  "why should I care" opener into real content; one essay = one
  complete payoff. Proven across Phases 2-6.
- **Simulator stays at 4 qubits** — Hard cap (REQ-13) survived
  6 v3 algorithm phases. Qiskit export handles the only algorithm
  exceeding 4: Shor on N=15.
- **Qiskit export as Shor's pedagogical bridge** — Stronger framing
  than watered-down in-browser Shor: "you've learned the parts; now
  run the real thing." Shipped on every essay's `CircuitView` and
  sandbox toolbar from Phase 1 onward.
- **One essay per phase, foundation-first** — v1 lesson honored;
  v3 essays are denser than v1's so one-per-phase rhythm fit.
- **Flagship = Teleportation (Phase 2)** — Extended v1's
  entanglement story most directly; lowest pedagogical risk for
  format validation.
- **`MultiBlochPanel` renders mixed states inside the sphere** —
  Truthful visualization; sphere only honestly shows pure
  single-qubit states.
- **VQE classical optimizer in vanilla TS** — Same inspectability
  bar as the simulator; ~50-100 LOC. Zero new runtime dependencies.
- **Qiskit export ships everywhere, not just on Shor** — Strengthens
  "now go do this for real" theme across all essays; sibling to v1's
  "remix in sandbox" CTA.
- **Progress indicator is localStorage-only (not tracking)** —
  Honors "no analytics. ever." with a one-line PROJECT clarification.
- **Mirror discipline (D-45)** — Same-commit updates to
  nav-graph.test, concept-map.test, sandbox-links.test,
  bundle-budget, sitemap when routes change. Held across all 7
  phases.
- **Concept-map FLAT layout ships** — Layout-audit verdict (OPS-03);
  14 nodes in four rows reads cleanly. Re-evaluate only on
  user-reported clutter.
