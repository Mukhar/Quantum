# Phase 1 — Foundation — Context

**Gathered:** 2026-06-29
**Status:** Ready for execution

## Phase Boundary

This phase ships the two cross-cutting infrastructure pieces every
later v3 phase depends on:

1. **Qiskit text export** — usable from both the sandbox toolbar
   (QSK-01) and every essay's `CircuitView` (QSK-02), with a golden
   gate-coverage test (QSK-03) that prevents drift as the simulator
   gate set evolves.
2. **Per-route bundle-size CI gate** — fails the build when a tracked
   route's JS bundle exceeds its declared ceiling (OPS-04). Required
   before v3 essays start landing — five new essays + per-essay
   widgets will otherwise creep the bundle silently.

Both pieces are pre-requisites for Phase 2 (Teleportation): the
flagship essay's `CircuitView` ships Qiskit export from day 1, and
the new essay route comes with a bundle ceiling in the CI manifest.

## Implementation Decisions

### Qiskit text export

- **One canonical module:** `src/lib/quantum/qiskit.ts` exports
  `toQiskit(circuit, opts?)` returning a Python source string. No
  separate "sandbox flavor" vs "essay flavor"; same module everywhere.
- **Output shape (fixed):**
  ```python
  # Generated by quantum-site v3 — circuit fragment <fragment>
  from qiskit import QuantumCircuit

  qc = QuantumCircuit(<n>, <n>)
  qc.h(0)
  qc.cx(0, 1)
  qc.rx(0.7853981633974483, 2)
  qc.measure(0, 0)
  ```
  Single `QuantumCircuit(n_qubits, n_qubits)` with classical register
  width = qubit count. `measure(q, q)` always uses the same index for
  both quantum and classical bit (matches simulator's per-qubit
  measurement semantics).
- **Gate mapping (locked):**

  | Simulator op (`circuit.ts`) | Qiskit instruction         |
  |-----------------------------|----------------------------|
  | `{kind:"gate",gate:"X"}`    | `qc.x(q)`                  |
  | `{kind:"gate",gate:"Y"}`    | `qc.y(q)`                  |
  | `{kind:"gate",gate:"Z"}`    | `qc.z(q)`                  |
  | `{kind:"gate",gate:"H"}`    | `qc.h(q)`                  |
  | `{kind:"gate",gate:"S"}`    | `qc.s(q)`                  |
  | `{kind:"gate",gate:"T"}`    | `qc.t(q)`                  |
  | `{kind:"gate",gate:"I"}`    | `qc.id(q)`                 |
  | `{kind:"cnot",control,target}` | `qc.cx(control, target)` |
  | `{kind:"rot",axis:"X",θ}`   | `qc.rx(theta, q)`          |
  | `{kind:"rot",axis:"Y",θ}`   | `qc.ry(theta, q)`          |
  | `{kind:"rot",axis:"Z",θ}`   | `qc.rz(theta, q)`          |
  | `{kind:"measure",qubit}`    | `qc.measure(q, q)`         |

- **Header comment includes the URL-codec fragment** so the reader
  can paste the link back into the sandbox; this completes the
  "remix here ↔ run for real" round-trip.
- **Empty steps emit no instructions.** Qiskit lacks an explicit
  "barrier between timeslices" semantic we care about; mirroring our
  step-grouped order is purely cosmetic. Don't bother.
- **Float formatting:** `theta.toPrecision(17)` (full f64). Don't try
  to recognize π/4 etc — Qiskit's `numpy.pi/4` import would balloon
  the snippet for negligible readability win.
- **No empty circuits, no `qubits=0`:** `toQiskit` calls
  `validateCircuit` first; errors bubble up as `CodecError` (already
  the codec's pattern).
- **No reverse-direction (Qiskit → sandbox).** Explicitly v4+.

### Sandbox toolbar button (QSK-01)

- **Reuses the share-button pattern** end-to-end:
  - Markup: `<button data-action="qiskit" data-tooltip="...">Copy as Qiskit</button>`
    inside the existing `[data-sandbox-toolbar]` element in
    `src/pages/sandbox/index.astro`.
  - Wiring: `composer.client.ts` queries
    `toolbar.querySelector("[data-action='qiskit']")` and attaches a
    click handler that calls a new
    `copyQiskitSnippet()` exported from `persistence.ts`.
  - The new function mirrors `copyShareUrl`: encode → clipboard →
    `showToast` success/error.
- **Position:** immediately after the "Copy share URL" button,
  before "Save". Keeps related copy-to-clipboard actions adjacent.

### `CircuitView` button (QSK-02)

- **`CircuitView` stays server-rendered (zero hydration cost).**
  Adds a single `<button>` element below the SVG (next to the
  existing "Remix in sandbox →" link).
- **Bake the Qiskit text at build time** — Astro frontmatter already
  has the `Circuit`; call `toQiskit(circuit)` and store the result
  in a `data-qiskit` attribute on the button. Avoids shipping the
  encoder to the client.
- **A single global hydrator** lives at
  `src/lib/circuitview/qiskitCopy.client.ts` (or as a tiny inline
  `<script is:inline>` inside `CircuitView.astro`). Attaches one
  click listener via event delegation:
  `document.addEventListener("click", ev => { if (ev.target.matches("[data-action='copy-qiskit']")) navigator.clipboard.writeText(ev.target.dataset.qiskit) })`.
  Total client JS budget for the feature: < 1 KB.
- **`showToast`-style confirmation** — reuse the sandbox toast
  function if cheap; otherwise emit a 1.5s inline `aria-live`
  announcement under the button. Decide during implementation; either
  works.

### Drift-proof gate coverage (QSK-03)

- **Two test families:**
  1. **Per-gate mapping table** — a `test.each` over the GATE MAP
     above asserts `toQiskit(singleOpCircuit(...))` contains the
     expected Qiskit substring.
  2. **Exhaustive coverage** — read the union of `DiscreteGate`,
     `RotAxis`, and `kind` values from `circuit.ts` and assert every
     one of them is present in the mapping. **This is the actual
     drift gate** — fails the test when a new simulator gate is
     added without a Qiskit mapping.
- **Output validity** — pick 3 canonical circuits (Bell pair,
  3-qubit GHZ, Deutsch oracle on `Rx(π)`) and snapshot their full
  Qiskit text. Hand-eyeball once; future changes prove
  regression-free.

### Per-route bundle CI gate (OPS-04)

- **A standalone Node script** at `scripts/check-bundle-budget.mjs`:
  1. Reads `bundle-budget.json` at the repo root (committed).
  2. After `astro build`, walks `dist/**/*.html` and, for each HTML
     file:
     - Extracts every `<script src="/_astro/...">` URL via a tiny
       regex (no JSDOM).
     - Sums the gzipped byte size of each referenced file under
       `dist/_astro/` using `zlib.gzipSync`.
     - Maps the HTML path to a "route key" (e.g.
       `dist/sandbox/index.html` → `sandbox`).
  3. Compares the per-route sum to the ceiling in `bundle-budget.json`.
  4. Exits 1 with a clear per-route over/under table on failure.
- **`bundle-budget.json` (initial):**
  ```json
  {
    "version": 1,
    "comment": "Gzipped JS bytes shipped per route (HTML script tags).",
    "routes": {
      "index":            8192,
      "qubit":            8192,
      "measurement":      8192,
      "superposition":    8192,
      "entanglement":     8192,
      "gates":            8192,
      "deutsch":          12288,
      "cnot-bell":        12288,
      "feedback":         8192,
      "feedback/thanks":  4096,
      "gallery":         24576,
      "sandbox":         98304
    }
  }
  ```
  Numbers are placeholders — the implementer should run a clean
  `npm run build`, record actual gzipped sizes, then add a **20%
  headroom** to each (ceiling = round_up(actual × 1.2, 1024)).
  Commit the actual ceilings in the same plan.
- **Script invocation:** `npm run check:bundle` → wired into
  `package.json` AND into a `pretest` step (so `npm test` runs it).
  CI runs `npm run build && npm run check:bundle && npm test`.
- **Failure UX:**
  ```
  Bundle budget exceeded:
    /sandbox        128.4 KB  > ceiling 96.0 KB  (+32.4 KB)
    /gallery         28.1 KB  >          24.0 KB  ( +4.1 KB)

  Either:
    - Trim the diff that introduced the regression, OR
    - Update bundle-budget.json ceilings explicitly in the same commit
  ```
- **Routes not in the manifest:** warn (not fail). The script lists
  un-budgeted routes at the bottom so they get a ceiling on the next
  commit. Prevents accidentally bypassing the gate by forgetting to
  add an entry.

### Discretion (executor decides)

- Whether to emit the leading comment block as one `#` line or three
  (cosmetic, < 0.1 KB delta).
- Whether the `CircuitView` Qiskit hydrator is a separate
  `.client.ts` file or an inline `<script is:inline>` element inside
  `CircuitView.astro`. Pick whichever ends up smaller after build.
- Whether `showToast` is reused for `CircuitView` (Astro layout
  coupling) or replaced with a local `aria-live` element.

## Specific Ideas

- The CircuitView baked `data-qiskit` attribute means the Qiskit text
  for `CircuitView` consumers is already in the HTML — view-source
  reveals it. Acceptable: it's the same Python the user is about to
  copy. Not sensitive.
- Bundle budget headroom of 20% is generous but matches how often we
  iterate on essay pages. Tighter (10%) means too many false-positive
  CI failures; looser (50%) defeats the gate's purpose.
- The Qiskit fragment-URL header comment is a small but real
  pedagogical lever: it lets the reader paste the URL into the
  sandbox AND get a runnable Qiskit snippet from the same starting
  state. Reinforces the "this is the same circuit, two surfaces"
  story.

## Deferred Ideas

- **Qiskit Aer simulator-specific exports** (e.g. `transpile` calls,
  noise models) — out of scope; users adding noise know how.
- **Reverse direction (Qiskit text → `Circuit`)** — explicitly v4+;
  not blocking.
- **Multi-circuit batch export** (gallery → notebook) — v4 gallery
  enhancement; would compose `toQiskit` per entry, but neither v3 use
  case demands it.
- **Per-essay bundle ceiling enforcement at PR-comment time** — the
  CI script fails the build; surface as a GitHub Action review
  comment is a v3.1 nice-to-have, not OPS-04.

In scope for Phase 1:
- `src/lib/quantum/qiskit.ts` (new) + tests
- `src/components/CircuitView.astro` (existing button + data attr)
- `src/pages/sandbox/index.astro` (toolbar markup)
- `src/lib/sandbox/composer.client.ts` (wire toolbar button)
- `src/lib/sandbox/persistence.ts` (`copyQiskitSnippet` export)
- Optional new tiny client module for `CircuitView` hydration
- `scripts/check-bundle-budget.mjs` (new)
- `bundle-budget.json` (new, repo root)
- `package.json` scripts (`check:bundle`, `pretest`)
- Tests under `tests/quantum/qiskit.test.ts` and any necessary essay-bundle drift check

Out of scope for Phase 1:
- Any v3 essay file (those land in Phases 2-6)
- Any change to the simulator gate set or matrices
- Any change to the URL codec
- CI runner config (GitHub Actions YAML, if any) — assumed `npm test`
  is already the CI command; if it isn't, the project hasn't shipped
  one yet and that's not Phase 1's job
