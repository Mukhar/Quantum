## Physics / math correctness verified end-to-end

For the canonical Shor target `N = 15, a = 7`:
- 8-qubit register split as 4 counting + 4 work; work register initialized to `|0001⟩` (representing integer 1, so the modular exponentiation starts from `7^0 mod 15 = 1`).
- Four controlled blocks apply `×7^1, ×7^2, ×7^4, ×7^8 mod 15` to the work register, gated on counting qubits `q0..q3`. Periodicity manifests as `7^4 ≡ 1 (mod 15)`.
- Inverse QFT decomposed textbook-style: per-counting-qubit Hadamard with `R†_π/2`, `R†_π/4`, `R†_π/8` controlled phase rotations and a final SWAP block reversing the register order.
- Four measurements on the counting register; expected continued-fractions post-processing recovers `r = 4` from a measured bin near a multiple of `16 / r`.
- Qiskit export uses `QFT(4, do_swaps=True).inverse()` so the Python on the page matches the canonical textbook reference users will compare against.

For RSACountdown correctness:
- `estimateLogicalQubits(2048) = 4104`, `estimateLogicalQubits(3072) = 6152`, `estimateLogicalQubits(4096) = 8200` (all verified in `rsa-countdown.test.ts`).
- Marker ordering invariant (`today < roadmap < break`) asserted by construction and tested.
- `aria-live="polite"` summary text branches on `pastThreshold = (available >= required)` and is verified for both branches.
